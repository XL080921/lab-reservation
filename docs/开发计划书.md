## 二、项目背景与目标

### 2.1 背景问题

高校及科研实验室普遍存在设备借用混乱的问题：

- **信息不透明**：师生不知道哪些设备可用、何时空闲
- **冲突频发**：多人同时借用同一设备，缺乏时间协调
- **审批缺失**：设备借用无登记、无审批，责任不清
- **损坏追责难**：设备损坏后难以追溯使用人
- **统计空白**：无法掌握设备利用率，采购决策无数据支撑

### 2.2 建设目标

| 目标 | 描述 | 衡量标准 |
|---|---|---|
| 规范借用流程 | 预约 → 审批 → 使用 → 归还全流程线上化 | 预约覆盖率 100% |
| 消除时间冲突 | 自动检测时间段重叠，拒绝冲突预约 | 冲突率降至 0 |
| 设备状态可视 | 可预约/使用中/维护中/损坏四态管理 | 实时状态准确率 ≥ 99% |
| 损坏可追溯 | 报损关联设备与使用人，责任明确 | 报损记录完整率 100% |
| 数据驱动决策 | 使用排行、审批通过率、状态分布统计 | 月度统计报表自动生成 |

| 字段 | 类型 | 说明 |
|---|---|---|

| model | TEXT | 型号 |

| location | TEXT | 存放位置 |
| status | TEXT | available / in_use / maintenance / damaged |
| description | TEXT | 详细描述 |

### 5.2 reservations（预约表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 自增主键 |
| equipment_id | INTEGER FK | 关联设备 |
| user_name | TEXT | 借用人姓名 |
| purpose | TEXT | 使用目的 |
| start_time | TEXT | 开始时间 |
| end_time | TEXT | 结束时间 |

### 5.3 approvals（审批表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 自增主键 |
| reservation_id | INTEGER FK | 关联预约 |
| admin_name | TEXT | 审批人 |
| action | TEXT | approve / reject |
| comment | TEXT | 审批意见 |
| created_at | TIMESTAMP | 审批时间 |

### 5.4 damage_reports（报损表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 自增主键 |
| equipment_id | INTEGER FK | 关联设备 |
| reservation_id | INTEGER FK | 关联预约（可选） |

### 5.5 静态版数据模型（localStorage）

与上述表结构一一对应，以 JSON 数组形式存储：

```javascript

  lab_equipment:    [{id, name, model, category, location, status, description}],
  lab_reservations: [{id, equipment_id, user_name, purpose, start_time, end_time, status, created_at}],
  lab_approvals:    [{id, reservation_id, admin_name, action, comment, created_at}],
  lab_damage_reports:[{id, equipment_id, reservation_id, reporter, description, severity, created_at}]
}
```

## 六、核心算法

### 6.1 时间冲突检测

```
输入: equipment_id, start_time, end_time, exclude_id
输出: 冲突的预约列表

算法:
  1. 查询 equipment_id 下所有 status ∈ {pending, approved} 的预约
  2. 排除自身预约 (exclude_id)
  3. 筛选满足: 已有预约.start < 新预约.end AND 已有预约.end > 新预约.start
  4. 返回冲突列表
```

### 6.2 设备状态流转

```
                 ┌──────────┐
    预约通过      │ available │  ← 归还完成
  ┌─────────────→│  (可预约)  │←─────────────┐
  │              └─────┬─────┘              │
  ▼                    │ 报损                │
┌──────────┐     ┌─────▼─────┐              │
│  in_use  │     │  damaged  │              │
│ (使用中)  │     │  (已损坏)  │              │
└────┬─────┘     └───────────┘              │
     │                                       │
     │ 归还完成                               │
     └───────────────────────────────────────┘
```
